# OrvixPanel v0.2.0 — Core Hosting Engine (Aegis)

**Release date:** 2026-06-07
**Codename:** Aegis
**Tag:** `v0.2.0-core-hosting-engine`
**Status:** live-smoked on Linux (WSL Ubuntu 26.04)

## What this release actually proves

This v0.2.0 release is the first build of OrvixPanel that
performs **real Linux work** end-to-end. v0.1.0 was API surface
and stubs. v0.2.0 provisions real system users, writes real
nginx vhosts and real php-fpm pools, and serves real HTTP
traffic on a real Linux box.

Every line below was either unit-tested, live-smoked, or
both. The full smoke matrix (16 steps) ran clean on a fresh
Ubuntu 26.04 install. No claims without evidence.

## What ships in v0.2.0

### Linux system integration
- `useradd` / `userdel` with per-account private group
- `groupadd` for the private group
- Home dir at `/var/lib/orvixpanel/homes/<username>` mode 0751
  (owner full, world-traversable so nginx can serve)
- `public_html/<domain>/` doc root mode 0755
- `SuspendAccount` (usermod -L) / `UnsuspendAccount` (usermod -U)
- `DeleteAccount` does pkill -u, sleeps, retries userdel -r
- `DiskUsed(path)` via `du -sb`
- `InodeCount(path)` via `find ... | wc -l`
- All OS-exec split `//go:build linux` / `//go:build !linux`
  so the package compiles on every platform

### Per-domain nginx vhost generation
- Pure-Go `GenerateNginxConfig(VHostConfig)` returns a complete
  server block with security headers, open_basedir, fastcgi_pass,
  log files, dotfile deny
- Written to `/etc/nginx/conf.d/orvix/<user>-<domain>.conf`
- `nginx -t` validates, `systemctl reload nginx` reloads

### Per-domain PHP-FPM pool generation
- Pure-Go `GenerateFPMPool(FPMConfig)` returns a `[orvix-... ]`
  pool with `pm=ondemand`, open_basedir, memory_limit, expose_php=off
- Written to `/etc/php/<ver>/fpm/pool.d/orvix-...conf`
- `php-fpm<ver> -t` validates, `systemctl reload php<ver>-fpm` reloads

### HTTP API (replaces v0.1.0 stubs)
- `POST   /api/v1/accounts`                       — real ProvisionAccount
- `GET    /api/v1/accounts`                       — list (tenant-scoped)
- `GET    /api/v1/accounts/:id`                   — detail + live usage
- `PUT/POST .../suspend|unsuspend`               — real usermod
- `DELETE /api/v1/accounts/:id`                   — real userdel -r
- `GET    /api/v1/accounts/:id/usage`             — live du + find
- `POST   /api/v1/accounts/:id/domains`           — full pipeline:
                                                      doc root + vhost + pool
                                                      + nginx -t + php-fpm -t
                                                      + reload nginx + reload php
- `GET    /api/v1/accounts/:id/domains`           — list
- `DELETE /api/v1/accounts/:id/domains/:domain`   — remove vhost + pool + dir

### Audit
- Every CreateAccount / CreateDomain / Suspend / Delete / etc.
  writes an audit row with the action, resource, IP, user.
- Audit chain is unchanged from v0.1.0; v0.2.0 just uses it.

## What does NOT ship in v0.2.0 (deferred to later)

- Cron job manager (NEXT_PHASE_PLAN still lists it; the
  interface is in the spec but no concrete API/handlers yet)
- Git deploy (atomic symlink swap is implemented in
  `hosting.AtomicSwap` but not exposed via the HTTP API yet —
  v0.2.1)
- Multi-version PHP support (only the env-detected version
  is written; per-domain PHP version is a v0.2.1 item)
- Production installer (smoke was run by hand; v0.2.1 ships
  `scripts/install.sh` that does the WSL setup)
- Docker port-80 workaround (operators on bare VPS get 80 for
  free; Docker on the host steals it in WSL — same constraint
  as in v0.1.0)

## Also in this commit (preview, not live-smoked at integration level)

The commit also contains the v0.3.0 **Enterprise Edition** API
hardening layer that was built in the prior session but never
live-smoked at the HTTP level:

- Encrypted license store (AES-256-GCM, master key from env)
- Real license expiry + read-only mode (423 on writes when
  license is locked)
- Custom RBAC roles (admin-defined, JSON permissions)
- API key auth (separate from JWT, long-lived, scoped)
- Audit log search + CEF export over syslog (file/UDP/TCP)
- Tenant-level quotas (per-tenant max accounts/users/api
  keys/secrets/etc.)
- Encrypted secrets vault (per-tenant AES-256-GCM)
- 54 unit tests across 6 packages, all green

**Caveat:** the v0.3.0 features have unit tests but were not
end-to-end live-smoked (no curl loop exercising the JSON API
of every endpoint). They are tagged `v0.3.0-api-hardening-preview`
in spirit but not on a git tag — only the v0.2.0 hosting
engine gets a tag in this release because only it passed the
full verification gate.

## How to verify on a clean Linux VPS

```bash
# 1. install deps
apt-get update && apt-get install -y nginx php8.5-fpm php8.5-cli

# 2. build
git clone https://github.com/reachfm/orvixpanel
cd orvixpanel
go build -ldflags="-s -w -X main.version=0.2.0" -o bin/orvixpanel ./cmd/orvixpanel

# 3. boot
ORVIX_ALLOW_DEV=1 ORVIX_DEV_LICENSE_EXPIRES_AT=2030-01-01 \
  ORVIX_MASTER_KEY=$(head -c 32 /dev/urandom | base64) \
  ORVIX_DATABASE_DSN=/var/lib/orvixpanel/data.db \
  ORVIX_SERVER_BIND_ADDR=0.0.0.0:8443 \
  ORVIX_FPM_VERSION=8.5 \
  ./bin/orvixpanel

# 4. include /etc/nginx/conf.d/orvix/*.conf from the main
#    http block in /etc/nginx/nginx.conf (one-time setup)

# 5. follow smoke-phase2.sh from this commit
```
